REST 원칙

- 리소스와 행위를 분리
- header 와 body 를 분리
    - entity 의 내용은 body
    - 서버가 취해야할 행동의 근거는 header
- API 의 버전을 관리해야 한다.
- 서버의 응답방식과 클라이언트의 요청 방식은 동일해야 한다.
    - e.g.
        - JSON 요청 시 응답은 JSON
