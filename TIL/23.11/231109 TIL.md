# GET

- URL 에 요청하려는 정보가 담겨있음
    - 따라서 비밀번호 같은 민감한 정보를 다루는데는 적합하지 않음
    - URL 길이 제한 때문에 보내려는 정보량의 제한이 존재 
- 브라우저가 Caching 할 수 있음

# POST

- 서버의 상태를 변경한다.
- message의 body 에 요청하려는 정보가 담겨있음
