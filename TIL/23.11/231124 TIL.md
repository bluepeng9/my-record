# DNS round robin

- 서버의 수 만큼 공인 ip 주소가 필요함
- 부하가 균등하게 분산되지 않는다.
    - 프록시 서버와 브라우저는 DNS 캐싱을 하기 때문
- 서버가 다운되도 확인이 불가능
